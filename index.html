    Question 5: Is Python code compiled or interpreted?

    Answer 5: Python is an interpreted language, which means that Python code is executed line by line, in a sequential manner, by the Python interpreter. When you run a Python program, the source code is first compiled into bytecode, which is a low-level representation of the code that can be executed by the Python interpreter. The bytecode is then interpreted by the Python interpreter, which executes the instructions and produces the output. Unlike compiled languages, such as C++ or Java, Python does not need to be compiled into machine code before it can be executed. This means that Python programs can be developed and run quickly, without the need for a lengthy compilation process. However, it's worth noting that some Python implementations, such as PyPy, do include a just-in-time (JIT) compiler, which can improve the performance of Python code by compiling parts of the code into machine code at runtime. This can make Python code run faster, especially for certain types of applications, such as numerical computation or machine learning.

    Question 14: How is memory managed in python?

    Answer 14: Memory management in Python is handled by the Python runtime itself through a mechanism called Automatic Memory Management or Garbage Collection. Python uses a dynamic memory allocation model, which means that memory is allocated and deallocated as needed during program execution. When a new object is created in Python, the interpreter dynamically allocates memory for it from the heap. When an object is no longer needed, Python automatically deallocates the memory it occupied. Python uses a technique called reference counting to keep track of which objects are in use and which are no longer needed. Each object in Python has a reference count, which is the number of references to that object from other parts of the program. When an object's reference count reaches zero, it is no longer needed, and Python deallocates its memory automatically. In addition to reference counting, Python also uses a garbage collector to deallocate objects that cannot be reached by the program. The garbage collector runs periodically in the background, reclaiming memory from objects that are no longer in use.

    Question 15: How to install python on Windows and set path variables?

    Answer 15: To install Python on Windows and set up path variables, you can follow these steps:

    1. Download the latest version of Python from the official Python website (https://www.python.org/downloads/windows/). Make sure to choose the appropriate version for your system (32-bit or 64-bit).
    2. Run the installer and follow the prompts to install Python. Make sure to select the option to add Python to the system PATH during installation.
    3. Once Python is installed, open the Windows Start menu and search for "Environment Variables".
    4. Click on "Edit the system environment variables" to open the System Properties dialog box.
    5. Click on the "Environment Variables" button to open the Environment Variables dialog box.
    6. Under "System Variables", scroll down and find the "Path" variable. Click "Edit" to modify it
    7. Add the path to the Python installation directory to the Path variable. The default path is "C:\PythonXX" (where "XX" represents the version number of Python installed). For example, if you installed Python version 3.10 in the default location, you would add "C:\Python310" to the Path variable.
    8. Click "OK" to save the changes and close the Environment Variables dialog box.
    9. Close and reopen any command prompt or terminal windows to ensure the new path settings are applied.
    You should now be able to run Python from the command prompt or terminal window by typing "python" and pressing enter.
